# PrimiHub 平台功能分析报告
## 基于菜单结构的完整功能说明

---

## 一、平台概述

**PrimiHub Platform** 是一个企业级**多方隐私计算协作平台**，允许多个机构在不泄露原始数据的前提下进行联合计算、数据分析和机器学习。

**核心价值**：数据可用不可见，实现数据价值的安全流通和协作。

---

## 二、平台菜单结构与功能详解

```
PrimiHub Platform

```

## 三、各菜单详细功能说明

### 🔍 1. 隐匿查询 (Private Information Retrieval)

**菜单路径**：`隐匿查询`
**权限代码**：`PrivateSearch`
**核心价值**：在不暴露查询关键词的情况下检索数据库内容

#### 功能模块

| 子菜单 | 路由 | 主要功能 | 使用场景 |
|--------|------|---------|---------|
| **隐匿查询列表** | `/privateSearch/list` | • 查看所有PIR查询任务<br>• 过滤：机构、任务名、关键词、时间<br>• 查看任务状态和结果 | 管理和追踪所有隐匿查询任务 |
| **发起查询** | `/privateSearch/task` | • 选择数据资源<br>• 输入查询关键词<br>• 提交查询任务 | 创建新的隐匿查询任务 |
| **查询详情** | `/privateSearch/detail/:id` | • 查看任务执行详情<br>• 下载查询结果<br>• 查看日志和状态 | 查看具体任务的执行情况 |

#### 典型应用场景
- 🏥 **医疗场景**：医院在不暴露患者姓名的情况下查询其他医院是否有该患者记录
- 🏦 **金融场景**：银行查询客户在其他机构的征信记录，但不暴露具体查询对象
- 🔐 **隐私保护**：任何需要查询但不能暴露查询内容的场景

---

### 🔒 2. 隐私求交 (Private Set Intersection)

**菜单路径**：`隐私求交`
**权限代码**：`PSI`
**核心价值**：多方计算数据交集，各方只能看到交集结果，看不到对方独有数据

#### 功能模块

| 子菜单 | 路由 | 主要功能 | 使用场景 |
|--------|------|---------|---------|
| **求交列表** | `/PSI/list` | • 查看所有PSI任务<br>• 过滤：机构、任务名、状态、时间<br>• 查看任务耗时和进度<br>• 任务操作（取消、重试、删除） | 管理所有隐私求交任务 |
| **创建求交** | `/PSI/task` | • **步骤1-基础信息**：任务名、输出格式、结果名<br>• **步骤2-数据引入**：选择发起方和协作方、配置数据<br>• **步骤3-实现方法**：选择PSI算法 | 发起新的求交任务 |
| **任务详情** | `/PSI/detail/:id` | • 查看任务配置<br>• 查看求交结果<br>• 下载结果文件<br>• 查看执行日志 | 查看任务执行详情和结果 |

#### 支持的PSI算法
- ECDH-PSI（椭圆曲线Diffie-Hellman）
- KKRT-PSI
- BC22-PSI
- 其他可配置算法

#### 典型应用场景
- 📱 **广告营销**：多个平台求交找到共同用户，进行精准营销
- 🏦 **反洗钱**：多个金融机构求交可疑账户名单
- 🔍 **风控**：多方联合识别欺诈用户，但不暴露各自的用户数据
- 👥 **去重**：多方数据去重，计算重复度，但不暴露原始数据

### 📋 3. 项目管理 (Project Management)

**菜单路径**：`项目管理`
**权限代码**：`Project`
**核心价值**：组织多方协作的隐私计算项目，统一管理资源和任务

#### 功能模块

| 子菜单 | 路由 | 主要功能 | 使用场景 |
|--------|------|---------|---------|
| **项目列表** | `/project/list` | • 两种视图：卡片视图/表格视图<br>• 过滤：机构、角色、项目名、状态<br>• 项目统计：全部/我发起/我协作<br>• 项目操作：查看、编辑、关闭 | 查看和管理所有项目 |
| **新建项目** | `/project/create` | • 填写项目基本信息<br>• 指定发起方并关联资源<br>• 添加协作方并分配资源<br>• 保存并发布项目 | 创建多方协作项目 |
| **项目详情** | `/project/detail/:id` | • 查看项目信息<br>• 查看参与机构和角色<br>• 查看关联的资源<br>• 管理项目任务<br>• 项目审批（协作方） | 管理项目的完整生命周期 |
| **创建任务** | `/project/detail/:id/createTask` | • 在项目内创建模型训练任务<br>• 配置任务参数<br>• 选择算法和组件 | 在项目中发起具体任务 |
| **任务详情** | `/project/detail/:id/task/:taskId` | • 查看任务执行状态<br>• 查看任务结果<br>• 下载输出文件 | 跟踪任务执行情况 |

#### 项目协作流程
```
发起方创建项目 → 邀请协作方 → 协作方审批 → 关联资源 → 创建任务 → 执行计算 → 查看结果
```

#### 典型应用场景
- 🏥 **多方联邦学习**：多家医院联合训练疾病预测模型
- 🏦 **联合风控建模**：多个银行联合建立信用评分模型
- 📊 **多方数据分析**：多个企业联合进行市场分析

### 🤖 4. 模型管理 (Model Management)

**菜单路径**：`模型管理`
**权限代码**：`Model`
**核心价值**：管理联邦学习模型的全生命周期

#### 功能模块

| 子菜单 | 路由 | 主要功能 | 使用场景 |
|--------|------|---------|---------|

| **模型详情** | `/model/detail/:id` | • 查看模型基本信息<br>• 查看组件配置<br>• 查看预测结果<br>• 查看训练历史<br>• 模型运行和推理 | 查看模型详细信息和性能 |

#### 支持的模型类型
- 逻辑回归（Logistic Regression）
- XGBoost
- 神经网络（Neural Network）
- 自定义算法组件

- **数据预处理组件**：缺失值处理、标准化、归一化
- **特征工程组件**：特征选择、特征转换
- **训练组件**：各类机器学习算法

#### 典型应用场景
- 🏥 **疾病预测模型**：基于多方医疗数据训练疾病诊断模型
- 💳 **信用评分模型**：多方联合训练信用评分算法
- 📈 **推荐系统**：多平台联合训练推荐模型

### 💾 6. 资源管理 (Resource Management)

**菜单路径**：`资源管理`
**权限代码**：`ResourceMenu`
**核心价值**：统一管理多方数据资源，是所有隐私计算的基础

#### 功能模块

| 子菜单 | 路由 | 主要功能 | 使用场景 |
|--------|------|---------|---------|
| **我的资源** | `/resource/list` | • 查看用户上传的所有资源<br>• 过滤：ID、名称、标签、类型、Y值、上传者<br>• 操作：上传、编辑、删除、预览<br>• 权限控制：设置资源可见性 | 管理本机构的数据资源 |
| **协作方资源** | `/resource/unionList` | • 查看合作机构共享的资源<br>• 申请资源访问权限<br>• 查看授权状态 | 浏览和使用合作方资源 |
| **可申请资源** | `/resource/availableResources` | • 查看可以申请授权的资源<br>• 提交资源使用申请<br>• 跟踪申请状态 | 发现和申请新的数据资源 |
| **衍生数据** | `/resource/derivedDataList` | • 查看隐私计算产生的衍生数据<br>• 包括：求交结果、查询结果等<br>• 管理衍生数据的使用权限 | 管理计算产生的结果数据 |
| **新建资源** | `/resource/create` | • **方式1-数据库连接**：配置数据库信息、选择表<br>• **方式2-文件上传**：上传CSV/Excel文件<br>• 配置字段信息（字段名、类型、是否Y值）<br>• 设置资源标签和可见性 | 上传和注册新的数据资源 |
| **编辑资源** | `/resource/edit/:id` | • 修改资源基本信息<br>• 更新字段配置<br>• 调整权限设置 | 更新已有资源信息 |
| **资源详情** | `/resource/detail/:id` | • 查看资源完整信息<br>• 查看字段列表和配置<br>• 预览数据内容<br>• 查看使用历史 | 详细了解资源情况 |

#### 支持的数据源类型
- **关系型数据库**：MySQL、PostgreSQL、Oracle、SQL Server
- **文件上传**：CSV、Excel、JSON
- **大数据平台**：Hive、Spark（扩展支持）

#### 资源字段配置
- **字段类型**：整数、浮点数、字符串、日期
- **Y值标记**：标识目标变量（用于机器学习）
- **字段属性**：主键、索引、可为空

#### 资源权限控制
- **私有**：仅本机构可见
- **项目内可见**：仅项目参与方可见
- **公开**：所有机构可申请使用

#### 典型应用场景
- 📊 **数据目录**：企业数据资产的统一目录管理
- 🔐 **数据授权**：细粒度的数据使用权限控制
- 🔄 **数据流转**：安全的跨机构数据协作

#### 典型应用场景

## 四、平台技术架构

### 4.1 权限控制体系

```
权限验证流程：
用户登录
    ↓
后端返回权限列表 (grantAuthRootList)
    ↓
存储到 localStorage (PER_KEY)
    ↓
前端根据 authCode 匹配路由 name
    ↓
动态生成可访问路由
    ↓
路由守卫验证
    ↓
渲染授权菜单
```

### 4.2 核心数据库表

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| `sys_user` | 用户表 | user_id, user_account, user_password, role_id_list |
| `sys_role` | 角色表 | role_id, role_name |
| `sys_auth` | 权限表 | auth_id, auth_code, auth_type, p_auth_id |
| `sys_ra` | 角色权限关联 | role_id, auth_id |
| `sys_ur` | 用户角色关联 | user_id, role_id |
| `sys_user_whitelist` | 白名单表 | whitelist_type, whitelist_value, status |
| `sys_operation_log` | 操作日志表 | user_id, operation_type, operation_module, request_params |

### 4.3 前后端交互

```
前端组件路径：
/views/
  ├── privateSearch/    # PIR隐匿查询
  ├── PSI/             # 隐私求交
  ├── project/         # 项目管理
  ├── model/           # 模型管理
  ├── reasoning/       # 推理服务
  ├── resource/        # 资源管理
  └── setting/         # 系统设置

API接口路径：
/data/
  ├── pir/            # PIR相关API
  ├── psi/            # PSI相关API
  ├── project/        # 项目管理API
  ├── model/          # 模型管理API
  ├── reasoning/      # 推理服务API
  └── resource/       # 资源管理API
/sys/
  ├── user/           # 用户管理API
  ├── role/           # 角色管理API
  ├── organ/          # 机构节点API
  ├── whitelist/      # 白名单API
  └── operationLog/   # 日志API
```
