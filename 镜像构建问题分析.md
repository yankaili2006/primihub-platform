# PrimiHub Platform 镜像构建问题分析与修复

## 问题分析

### 出现的问题
在尝试将 docker-all-in-one 中的 platform 镜像升级到本地构建的 1.8.0 版本时，所有容器不断重启，错误信息为：
```
/bin/sh: 1: java: not found
```

### 根本原因

**镜像混淆**：用户使用了**前端构建脚本**构建了 `192.168.99.10/primihub/platform:1.8.0`，但这个镜像只包含 Nginx + Vue.js 静态文件，**不包含 Java 运行环境**。

而 docker-all-in-one 中的 `PRIMIHUB_PLATFORM` 变量需要的是**后端 Java 镜像**，包含 application.jar 和 gateway.jar。

---

## 镜像对比

### 错误的构建方式 ❌

**使用的脚本**: `quick-build-platform.sh`

```bash
# 构建的是前端镜像
cd primihub-webconsole
npm install && npm run build:prod
docker build -t 192.168.99.10/primihub/platform:1.8.0 -f Dockerfile.local .
```

**Dockerfile 内容**:
```dockerfile
FROM nginx:1.20
COPY dist/ /usr/local/nginx/html/
```

**镜像内容**:
- ✅ Nginx Web 服务器
- ✅ Vue.js 静态文件
- ❌ **没有 Java**
- ❌ **没有 application.jar**
- ❌ **没有 gateway.jar**

**镜像用途**: 这个镜像应该用于 `PRIMIHUB_WEB_MANAGE`（前端），而不是 `PRIMIHUB_PLATFORM`（后端）！

---

### 正确的构建方式 ✅

**使用的脚本**: `build-platform-backend.sh` (新创建的修复版)

```bash
# 1. 编译 SDK
cd primihub-sdk
mvn clean install -Dmaven.test.skip=true

# 2. 编译 Service
cd primihub-service
mvn clean install -Dmaven.test.skip=true

# 3. 构建镜像
docker build -t 192.168.99.10/primihub/platform:1.8.0 \
    -f ./primihub-service/Dockerfile.ubuntu \
    ./primihub-service
```

**Dockerfile 内容** (Dockerfile.ubuntu):
```dockerfile
FROM ubuntu:22.04

# 安装 OpenJDK 8
RUN apt-get update && \
    apt-get install -y openjdk-8-jre-headless tzdata && \
    ln -fs /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

# 复制 JAR 文件
ADD application/target/*-SNAPSHOT.jar /applications/application.jar
ADD gateway/target/*-SNAPSHOT.jar /applications/gateway.jar

WORKDIR /applications
ENTRYPOINT ["/bin/sh","-c","java -jar -Dfile.encoding=UTF-8 /applications/application.jar"]
```

**镜像内容**:
- ✅ Ubuntu 22.04 系统
- ✅ OpenJDK 8 JRE
- ✅ application.jar (165MB)
- ✅ gateway.jar (124MB)

**镜像用途**: 这个镜像用于 `PRIMIHUB_PLATFORM`（后端），提供 Java 应用服务。

## 常见错误和解决方案

### 错误 1: "java: not found"

**原因**: 使用了前端构建脚本，镜像中没有 Java

**解决**: 使用 `build-platform-backend.sh` 重新构建

---

### 错误 2: "JAR 文件不存在"

**原因**: Maven 编译失败或未执行

**解决**:
```bash
# 手动编译
cd ~/github/primihub-platform/primihub-service
mvn clean install -Dmaven.test.skip=true

# 检查 JAR 文件
ls -lh application/target/*.jar
ls -lh gateway/target/*.jar
```

---

### 错误 3: "镜像名称混淆"

**问题**: 不确定应该构建哪个镜像

**说明**:
- **前端镜像** (primihub-web): Nginx + Vue.js，用于 PRIMIHUB_WEB_MANAGE
  - 构建: `quick-build-platform.sh`
  - 大小: ~224MB

- **后端镜像** (primihub-platform): Java + application.jar + gateway.jar，用于 PRIMIHUB_PLATFORM
  - 构建: `build-platform-backend.sh` ✅
  - 大小: ~1.1GB

## 总结

### 问题根源
用户错误地使用了**前端构建脚本** (`quick-build-platform.sh`) 来构建 `platform` 镜像，但这个镜像只包含 Nginx 和静态文件，不包含 Java 和后端 JAR 文件。

### 解决方案
创建了新的 `build-platform-backend.sh` 脚本，正确地：
1. ✅ 编译 Java 项目 (SDK + Service)
2. ✅ 使用包含 Java 的 Dockerfile (Dockerfile.ubuntu)
3. ✅ 构建包含 application.jar 和 gateway.jar 的镜像
4. ✅ 验证镜像包含 Java 和 JAR 文件

### 关键要点
- **前端镜像** (primihub-web): Nginx + Vue.js → 用于 PRIMIHUB_WEB_MANAGE
- **后端镜像** (primihub-platform): Java + JAR → 用于 PRIMIHUB_PLATFORM
- 不要混淆两种镜像的构建脚本！
