## 🔍 详细排查步骤（请严格按照步骤操作）

### 步骤1：打开开发者工具

1. 在浏览器中打开系统：http://100.64.0.23:30811/
2. 按 **F12** 键（Mac: **Cmd + Option + I**）
3. 确认开发者工具已打开

### 步骤2：切换到Network标签

1. 点击开发者工具顶部的 **Network** 标签
2. 确保 **Preserve log** 选项已勾选（保留日志）
3. 点击 **清空按钮**（🚫图标）清空现有日志

### 步骤3：触发请求

1. 在左侧菜单中点击 **白名单管理** 或 **白名单列表**
2. 等待页面加载或出现错误提示

### 步骤4：查看Network请求（最关键！）

在Network列表中找到 **findWhitelistPage** 请求（如果没有找到，说明根本没发送请求）

#### 如果找到了请求：

点击该请求，查看右侧详情面板，提供以下信息：

**Headers 标签：**
```
Request URL: （复制完整URL，例如：http://100.64.0.23:30811/prod-api/whitelist/findWhitelistPage?...）
Status Code: （状态码，例如：404 Not Found 或 200 OK）
Request Method: （请求方法，例如：GET）
```

**Response 标签：**
```
（复制完整的响应内容）
例如：
{"msg":"xxx","code":xxx}
或
<!DOCTYPE html>...
```

**Query String Parameters 标签：**
```
pageNum: 1
pageSize: 10
timestamp: xxxxx
nonce: xxx
token: xxxxx (复制token的前20个字符)
```

#### 如果没找到请求：

说明前端根本没有发送API请求，请检查：

1. **Console 标签** - 是否有JavaScript错误（红色文字）？复制所有错误信息
2. **前端路由** - 浏览器地址栏的URL是什么？（应该是 `.../#/whitelist/list`）

### 步骤5：检查Console错误

切换到 **Console** 标签，查看是否有红色错误信息：
- 复制所有错误信息
- 特别注意以下类型的错误：
  - `404 Not Found`
  - `Failed to load resource`
  - `ERR_CONNECTION_REFUSED`
  - JavaScript运行时错误

## 🎯 根据不同情况的解决方案

### 情况A：Network中有findWhitelistPage请求

#### A1: Status Code = 404
**可能原因**：
- Nginx转发配置问题
- Gateway路由问题
- Application服务未启动

**需要信息**：Response标签的完整内容

#### A2: Status Code = 200，但Response是错误
**可能原因**：
- Token无效
- 权限不足
- 参数缺失

**需要信息**：Response内容中的code和msg

#### A3: Status Code = 503
**可能原因**：Application服务不可用

**解决方案**：后端已确认Application正常，需要重新登录

### 情况B：Network中没有findWhitelistPage请求

**可能原因**：
1. 前端JavaScript加载失败
2. 前端路由匹配错误
3. Vue组件未正确加载
4. 权限控制阻止了API调用

**需要信息**：
- Console中的错误信息
- 浏览器地址栏的完整URL
- Application标签 → Local Storage → token的值（前20字符）

## 💡 常见问题

### Q1：我点击白名单后立即显示404，没有加载过程
**A**：这是前端路由404，不是API 404。可能：
- 前端代码未正确部署
- 浏览器缓存了旧版本
- 路由配置问题

### Q2：页面能打开，但数据显示404
**A**：这是API调用404。查看Network标签的详细信息。

### Q3：显示"请求出错"但没有具体错误码
**A**：可能是网络错误或CORS问题。查看Console标签。
