⚠️  重要提示
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

由于PrimiHub的安全设计,getPubKey接口需要token认证,导致无法直接
通过API从零开始获取token。

解决方案: 首次从浏览器获取token,之后就可以一直使用API了。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第一步: 从浏览器获取Token
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Node0 - gateway0】

1. 浏览器访问: http://172.20.0.12:8080
2. 登录账号: admin  密码: 123456
3. 按 F12 打开开发者工具
4. 切换到 Network(网络) 标签
5. 刷新页面(F5)
6. 点击任意API请求(如 getOrganList)
7. 右侧 Headers → Request Headers → 复制 token 字段值

【Node1 - gateway1】

1. 浏览器访问: http://172.20.0.2:8080
2. 登录账号: admin  密码: 123456
3. 重复上面相同的步骤获取token

🎯 方法1: 交互式创建(推荐新手)

cd /home/primihub/primihub-platform/primihub-test-framework/tests

python3 create_resources_interactive.py

# 然后按照提示操作:
# 1. 输入 y (已准备好token)
# 2. 粘贴Node0的token
# 3. 选择要创建的数据集(如: 1,2)
# 4. 决定是否在Node1也创建
# 5. 粘贴Node1的token
# 6. 选择数据集
# 7. 查看创建结果


🎯 方法2: 批量创建(适合重复使用)

cd /home/primihub/primihub-platform/primihub-test-framework/tests

# 1. 编辑脚本
vim add_resources_to_nodes.py

# 2. 找到第23和27行,填入token:
NODE0_TOKEN = "你的node0_token"
NODE1_TOKEN = "你的node1_token"

# 3. 保存后运行
python3 add_resources_to_nodes.py

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q: Token验证失败怎么办?
A: Token可能已过期(24小时),重新从浏览器获取新token

Q: 找不到token字段?
A: 确保已登录,并查看API请求的Request Headers,不是Response

Q: 创建资源失败?
A: 1. 检查token是否有效
   2. 确认用户有创建资源的权限
   3. 查看详细错误信息

Q: 可以只在一个节点创建吗?
A: 可以,在交互式脚本中选择n跳过另一个节点
   或在批量脚本中只填一个节点的token
