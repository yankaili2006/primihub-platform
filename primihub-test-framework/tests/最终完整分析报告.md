```
      |                            |                           |

```

**当前问题链**:
1. ❌ Meta服务的gRPC端口未正确配置/启动
2. ❌ PrimiHub Node无法连接Meta服务
3. ❌ 资源无法注册到数据集
4. ❌ 资源创建API返回"请求异常"

## 💡 完整解决方案

### 方案1: 配置Meta服务gRPC (推荐但需要更多信息)

Meta服务需要配置gRPC端口。可能需要:

1. **检查Meta服务配置**:
   ```bash
   docker exec primihub-meta2 env | grep -i grpc
   # 或查看配置文件
   ```

2. **修改Nacos配置** (如果meta服务从Nacos读取配置):


## 🎓 技术发现

### 1. PrimiHub架构层次
```
Web层 (Vue.js)
  ↓
API层 (Spring Boot - application容器)
  ↓
业务层 (Java Service)
  ↓
gRPC层 (Java → C++)
  ↓
计算层 (PrimiHub Node - C++)
  ↓
元数据层 (Meta Service)
```

### 2. 资源创建完整流程
```
1. 文件上传 → fileId
2. API参数验证 (8个必需字段)
3. 文件内容分析 (行数/列数/MD5)
4. 生成resourceFusionId
5. ⚠️ gRPC注册到Node (关键步骤)
6. Node注册到Meta服务 (关键步骤)
7. 数据库持久化
8. 消息队列通知
9. 返回成功
```

### 3. gRPC配置依赖关系
```
Application → grpcClient.address = "primihub-node2"
                       .port = 50052

Node → meta_service.ip = "primihub-meta2"
            .port = 9099 (配置的)
            .port = 8080 (实际的?)
            .port = ??? (正确的?)
```

### 4. 字段类型映射
```
API          Java        CSV

"Integer" →  Integer  →  numeric
"Double"  →  Double   →  decimal
```
