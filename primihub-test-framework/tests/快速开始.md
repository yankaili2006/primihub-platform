# 🚀 联邦学习测试 - 快速开始（使用Token）

## 📝 准备工作

### 1. 获取Token

**方法：从浏览器开发者工具获取**

1. 打开浏览器，访问PrimiHub平台并登录
2. 按 `F12` 打开开发者工具
3. 切换到 **Network**（网络）标签
4. 刷新页面或点击任意菜单
5. 点击任意API请求（如 `getOrganList`）
6. 在右侧找到 **Headers** → **Request Headers**
7. 找到 `token` 字段，复制其完整值

Token示例：
```

```

测试联邦学习的完整流程，包括创建数据、项目等。

## 🔧 常见问题

### Q1: Token验证失败

**原因：**
- Token已过期
- Token复制不完整
- 服务地址错误

**解决：**
1. 重新登录获取新token
2. 确保复制完整的token（包括所有字符）
3. 检查 `BASE_URL` 是否正确（当前是 `http://172.20.0.12:8080`）

### Q2: 机构数量少于2个

**原因：**
联邦学习需要至少2个参与方

**解决：**
在PrimiHub系统中先创建至少2个机构

### Q3: 创建资源失败

**可能原因：**
- Token权限不足
- 资源数据格式错误
- 系统配置问题

**解决：**
查看错误信息中的具体原因，检查日志

### Q4: 找不到reports目录

**解决：**
```bash
# 创建reports目录
mkdir -p /home/primihub/primihub-platform/primihub-test-framework/tests/reports
```

## 💡 进阶使用

### 自定义测试数据

编辑 `test_with_token.py`，找到 `test_create_resources` 方法，修改资源数据：

```python
{
    "resourceName": "我的数据集",
    "resourceDesc": "自定义描述",
    "resourceRowsCount": 5000,  # 修改行数
    "resourceColumnCount": 10,  # 修改列数
    # ...
}
```

### 调整日志级别

在测试脚本末尾修改：

```python
logging.basicConfig(
    level=logging.DEBUG,  # INFO改为DEBUG，查看更详细日志
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
```

### 单独测试某个步骤

注释掉不需要的测试：

```python
def run(self):
    # self.test_verify_token()
    # self.test_get_organs()
    self.test_create_resources()  # 只测试创建资源
    # ...
```
