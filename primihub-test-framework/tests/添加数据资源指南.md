# 在node0和node1添加数据资源 - 使用指南

## 📝 概述

这个脚本可以通过API在node0和node1两个节点上自动创建数据资源，模拟真实的多方联邦学习场景。

## 🎯 创建的数据资源

### Node0 数据资源

1. **用户特征数据**
   - 用户ID、年龄、性别、城市、收入、学历
   - 10,000行数据
   - 适用于用户画像分析

2. **交易记录数据**
   - 用户ID、交易ID、金额、时间戳、商户ID、类别
   - 50,000行数据
   - 适用于交易行为分析

### Node1 数据资源

1. **信用特征数据**
   - 用户ID、信用分数、贷款次数、逾期次数、总负债、标签
   - 8,000行数据
   - 适用于信用风险评估

2. **交易记录数据**
   - 用户ID、交易ID、金额、时间戳、商户ID、类别
   - 50,000行数据
   - 适用于交易行为分析

## 🚀 快速开始

### 第一步：获取node0和node1的Token

**获取node0的token：**

1. 在浏览器中访问node0的网关地址（例如：http://172.20.0.12:8080）
2. 登录系统
3. 按F12打开开发者工具
4. 切换到Network标签
5. 刷新页面或执行任意操作
6. 点击任意请求，查看Headers中的token字段
7. 复制完整的token值

**获取node1的token：**

重复上述步骤，访问node1的网关地址（例如：http://172.20.0.2:8080）

### 第二步：配置脚本

编辑脚本，填入token：

```bash
cd /home/primihub/primihub-platform/primihub-test-framework/tests

vim add_resources_to_nodes.py
```

找到配置区域（约第14-19行），填入token：

```python
# node0 配置
NODE0_URL = "http://172.20.0.12:8080"  # gateway0的地址
NODE0_TOKEN = "你的node0_token"        # 填入node0的token

# node1 配置
NODE1_URL = "http://172.20.0.2:8080"   # gateway1的地址
NODE1_TOKEN = "你的node1_token"        # 填入node1的token
```

保存退出（vim中按Esc，然后输入`:wq`）

### 第三步：运行脚本

```bash
python3 add_resources_to_nodes.py
```

## 🔧 自定义配置

### 修改节点URL

如果你的节点地址不同，修改脚本中的URL：

```python
NODE0_URL = "http://你的node0地址:端口"
NODE1_URL = "http://你的node1地址:端口"
```

### 自定义数据集

在脚本的 `main()` 函数中，修改要创建的数据集：

```python
# node0创建的数据集类型
node0_datasets = ["user_features", "transaction_records"]

# node1创建的数据集类型
node1_datasets = ["credit_features", "transaction_records"]
```

可选的数据集类型：
- `user_features` - 用户特征数据
- `transaction_records` - 交易记录数据
- `credit_features` - 信用特征数据

### 添加新的数据集类型

在 `create_sample_resource_data()` 函数中添加新的数据集定义：

```python
datasets = {
    "your_dataset": {
        "name": f"{node_name}_你的数据集_{timestamp}",
        "desc": "数据集描述",

            {"columnName": "字段1", "columnType": "string", "columnDesc": "描述1"},
            {"columnName": "字段2", "columnType": "int", "columnDesc": "描述2"},
        ],
        "rows": 1000
    }

```

## ❓ 常见问题

### Q1: Token验证失败

**原因：**
- Token已过期
- Token复制不完整
- 节点地址错误

**解决：**
1. 重新登录获取新token
2. 确保复制完整的token
3. 检查节点URL是否正确

### Q2: 只想在一个节点上创建资源

**解决：**

只填入一个节点的token，另一个留空：

```python
NODE0_TOKEN = "你的token"
NODE1_TOKEN = ""  # 留空，跳过node1
```

### Q3: 创建资源失败

**可能原因：**
- 权限不足
- 资源名称重复
- 数据格式错误

**解决：**
- 查看错误信息
- 确认用户有创建资源的权限
- 检查资源数据格式

### Q4: 如何验证资源是否创建成功？

**方法1：** 查看脚本输出的验证信息

**方法2：** 登录Web界面查看数据资源列表

**方法3：** 查看生成的JSON文件
