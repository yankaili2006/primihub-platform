## 🔧 已修复的问题

### 问题1: 菜单权限配置格式不正确
**问题**: 新菜单的full_path使用了完整路径 `/setting/xxx`，而不是数据库ID格式
**解决**: 已修正为 `1029,xxxx` 格式（与现有菜单一致）
**结果**: ✅ 菜单可以正常显示在左侧导航栏

### 问题2: 白名单按钮权限类型错误
**问题**: 按钮权限被配置为菜单类型(auth_type=2)
**解决**: 已修正为按钮类型(auth_type=3)
**结果**: ✅ 白名单列表页面显示"新增白名单"按钮

## ❗ 故障排查

### 如果菜单不显示
1. **检查权限分配**
   ```sql
   SELECT a.auth_code, a.auth_name, COUNT(ra.id) as assigned
   FROM sys_auth a
   LEFT JOIN sys_ra ra ON a.auth_id = ra.auth_id AND ra.role_id = 1
   WHERE a.auth_code LIKE '%Management' OR a.auth_code LIKE '%Workflow' OR a.auth_code LIKE '%Log'
   GROUP BY a.auth_code, a.auth_name;
   ```

2. **清除Redis缓存**
   ```bash
   docker exec redis redis-cli -a primihub FLUSHALL
   ```

3. **检查用户角色**
   确保登录用户是超级管理员角色(role_id=1)

### 如果页面报404错误
1. 检查前端文件是否存在
   ```bash
   ls -la /home/primihub/primihub-platform/primihub-webconsole/src/views/setting/
   ```

2. 重新加载前端容器
   ```bash
   docker restart manage-web0 manage-web1 manage-web2
   ```
