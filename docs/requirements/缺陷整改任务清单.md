# 隐私计算数据可信共享 —— 缺陷整改任务清单

> 来源：`隐私计算数据可信共享缺陷问题.docx`（44 条验收缺陷）
> 方法：逐条对照 `primihub-webconsole`（前端）+ `primihub-service`（后端 controller/gateway/service/mapper/schema）代码定位根因
> 日期：2026-07-13

---

## 一、结论：44 条缺陷收敛为 10 类根因

绝大多数缺陷**不是孤立 bug**，而是 10 类系统性问题的重复表现。按修复性价比排序：

| 类 | 根因 | 影响缺陷数 | 修复面 | 工作量 |
|---|---|---|---|---|
| R1 | 前端成功判断字段写错 `res.returnCode==='0'`（应为 `res.code===0`） | ~7 | 前端多个 .vue | 小（批量改） |
| R2 | 前端页面是 mock 空壳（setTimeout+写死数据，从不调后端） | ~20 页 | 前端接线 | 中（后端多已就绪） |
| R3 | 网关缺路由 `/federatedLearning/**`、`/singleParty/**` | 联邦学习整块 | gateway yaml + Nacos | 小 |
| R4 | 权限种子数据未入库（按钮/菜单 authCode 缺失） | 白名单/租户/求差 | SQL 种子 | 小 |
| R5 | Schema 双轨割裂（`script/*.sql` 未并入启动建表 `schema-mysql.sql`） | 日志定义/数据需求/共享数据集 | SQL + 部署 | 小 |
| R6 | 前后端字段契约不匹配（键名/类型对不上） | 监控告警/时间戳/接口 | 前后端对齐 | 中 |
| R7 | 前端成功回调缺 else/catch → 失败静默"无响应" | 用户/项目/接口/时间戳新增 | 前端补分支 | 小 |
| R8 | 前后端接口路径不一致 → 404 | 租户隔离 ×2 | 前端改路径 | 小 |
| R9 | 时区：`@JsonFormat` 缺 `timezone="GMT+8"` | 角色创建时间 | 后端 1 处 | 极小 |
| R10 | 后端占位实现（直接 return success，不查库/不产文件） | 接口日志导出/求差日志导出 | 后端补实现 | 中 |

> ⚠️ 重要澄清：**联邦求差、警务数据融合、电子证照** 三个"整模块不存在/不可用"的缺陷，经核实**后端 Controller/Service/Repository/建表 SQL 全部已实现**，真因是 R2（前端空壳）+ R4（权限未授）+ R3（网关），**不是需要从零补开发**。这大幅降低整改工作量。

## 二、整改任务（按根因分组，含验收缺陷映射）

### 任务 T1 — 【R1】批量修正前端成功判断字段 `returnCode` → `code`
**根因**：后端统一响应体 `BaseResultEntity` 只有整型 `code`（成功=0），无 `returnCode` 字段。前端新页误判 `res.returnCode==='0'` 恒为 false，导致：后端**成功也弹红色"请求成功"错误**、或短路到 mock 假数据。
**改动文件**（把文件内所有 `returnCode === '0'` 改为 `code === 0`）：
- `src/views/setting/accessManagement.vue:404`
- `src/views/setting/cooperation.vue:587`
- `src/views/setting/approval.vue:468`
- `src/views/setting/dataExchange.vue:334`
- `src/views/project/ledgerExport.vue:366,468,538,614`
- `src/views/resource/requirementList.vue`（fetchData / handleSubmit）
- `src/views/resource/sharedDatasetList.vue`（fetchData / handleSubmit）
**覆盖缺陷**：接入方新增异常、合作方"一直请求成功"、审批工作流异常、数据交换"请求成功的异常"、台账导出、数据需求新增+模糊查询、共享数据集新增+模糊查询
**置信度**：高

### 任务 T2 — 【R2】前端 mock 空壳页接入真实接口
**根因**：以下页面用 `setTimeout`+写死数组模拟，`handleExport/handleDownload/fetchData` 从不调已定义好的 API，导致"导出提示成功但无文件""搜索框失效返回全量"。**后端接口大多已存在**。
**改动**（把 mock 替换为对应 `@/api/*` 真实调用 + blob 下载）：
- 联邦统计：`src/views/federatedStatistics/{resultExport,logRecord,logExport}.vue` → 调 `federatedStatisticsApi.js` 的 `exportStatisticsResult/downloadResultExportFile/getStatisticsLogs/exportStatisticsLogs`
- 联邦学习：`src/views/federatedLearning/{index,logRecord,logExport,dataMerge}.vue` → 调 `api/federatedLearning.js`（`getTaskList/getFederatedLearningLogs/exportFederatedLearningLog`）
- 警务融合：`src/views/policeDataFusion/` 7 个空壳页（`policeConnect/batchExchange/dataDecrypt/encryptedCompute/modelEncrypt/logRecord/logExport`）→ 接 `@/api/scene`（后端 `SceneController` 已实现）
- 电子证照：`src/views/electronicCert/` 8 个空壳页 → 接 `@/api/scene`
**覆盖缺陷**：联邦统计结果导出/日志记录/日志导出、联邦学习日志记录/日志导出/单方合并、警务数据融合整块、电子证照整块
**参考**：真实 blob 导出实现可直接复用 `src/views/FederatedLearning/list.vue:382-390`
**置信度**：高｜**工作量**：中（页面多，但后端就绪，纯接线）

### 任务 T3 — 【R3】网关补联邦学习/单方路由
**根因**：`primihub-service/gateway/src/main/resources/application.yaml` 用逐前缀显式路由、无 catch-all，**缺 `/federatedLearning/**` 与 `/singleParty/**`**，后端 Controller 已实现却被网关 404。
**改动**：仿现有 `/federatedAnalysis` 段，新增两条路由（含 StripPrefix/SysAuth/SysLog filter），并同步到 Nacos 上同名 gateway config。
**覆盖缺陷**：联邦学习整块"请求错误/加载任务列表失败"、单方数据其它子功能不可用
**置信度**：高｜**工作量**：小

### 任务 T4 — 【R4】补齐权限种子数据（按钮/菜单 authCode）
**根因**：前端按钮/菜单由后端 `grantAuthRootList` 的 authCode 过滤显示，相关 authCode 未入生产库。
**改动**：
- 白名单按钮：执行 `primihub-service/script/whitelist_button_permissions.sql`（3 个 privacy 库），并把 `WhitelistAdd/Edit/Delete` 并入主种子 `data-mysql.sql`+`data-complete-h2.sql`
- 租户：**全仓无任何 `Tenant*` 权限记录**，需新建 `tenant_button_permissions.sql`（一级菜单+`TenantList`+`TenantAdd/Edit/Delete/Freeze`）并授 role_id=1
- 联邦求差：执行 `init_permissions.sql`(求差段) 或 `fix_missing_auth_entries.sql`，授权 `Difference*` 到角色
- 收尾：清 Redis 权限缓存 → 重新登录
**覆盖缺陷**：白名单无新增按钮、租户无新增按钮、联邦求差"模块不存在"×3
**置信度**：高｜**工作量**：小
**附带修复**：`init_permissions.sql` 单方菜单用 `/SingleParty/*`（大写）与 `router/index.js` 的 `/singleParty/*`（小写）不一致，需统一。

### 任务 T5 — 【R5】统一 schema，把 `script/*.sql` 建表并入启动建表
**根因**：应用启动只跑 `schema-mysql.sql`/`schema-complete-h2.sql`，而下列表只定义在未接入启动的 `script/*.sql`，生产库缺表 → INSERT/SELECT 抛 SQLException → 500/"请求异常"。
**缺失表**：
- 三张 `sys_*_log_definition`（操作/调度/计算日志定义）+ `sys_schedule_log`/`sys_compute_log`（`script/log_management.sql`）
- `data_requirement`（`script/data_requirement.sql`，`user_id BIGINT NOT NULL`）
- `shared_dataset`（仅 `test-tools/init-privacy-db-tables.sql` 有，主 schema 无建表）
**改动**：把以上 `CREATE TABLE IF NOT EXISTS` 合并进 `schema-mysql.sql` 与 `schema-complete-h2.sql`（H2 去反引号/ENGINE），对现网执行 DDL。
**覆盖缺陷**：操作/调度/计算日志定义新增异常、数据需求新增异常、共享数据集新增异常
**置信度**：高｜**工作量**：小

### 任务 T6 — 【R6】对齐前后端字段契约
**根因**：前端提交字段名/类型与后端读取不一致，后端转换抛异常被吞。
**改动**：
- **监控告警**（覆盖 CPU/内存/磁盘/DB/JVM/Redis 6 条）：`MonitorService.java:270,273` 对 `enabled`(布尔) 用 `Integer.valueOf` 崩溃。后端改布尔/枚举容错；前端 `views/monitor/index.vue:598-605` 字段映射为后端契约：`enabled`→`isEnabled`(0/1)、`level`→`alertLevel`(int)、`notifyMethods[]`→`notifyMethod`(逗号串)、`notifyTargets`→`notifyTarget`；`getAlertConfig` 前后端加 `type` 参数以按类型回填。
- **时间戳**（存证）：`views/evidence/timestamp.vue` 的 `submitApply` 用 FormData 但 API 声明 `type:'json'` 且未传 `evidenceId`，后端 `EvidenceController.applyTimestamp` 只认 `evidenceId` → LACK_OF_PARAM。对齐表单与接口语义（文件走 multipart 或先建存证拿 id）。
- **接口新增**：前端 `apiForm.method` 与后端 `apiMethod`（`ApiManageService.java:63`）键名不符。
**覆盖缺陷**：监控 6 条、时间戳提交无响应、接口新增无响应
**置信度**：高（监控约 90%）｜**工作量**：中

### 任务 T7 — 【R7】前端成功回调补 else/catch，消除"无响应"
**根因**：`if (res.code===0){...}` 无 else、`.catch` 不提示，后端返回非 0 时页面静默 → 用户感知"点了没反应"。
**改动**（补 `else { this.$message.error(res.msg||'操作失败') }` + catch 提示）：
- `views/setting/user.vue:295 enterUserDialog`（另需修角色 `el-select` 漏 `multiple`）
- `views/project/create.vue:180-191 submitForm`
- `views/api/list.vue:436-450 submitForm`
- `views/evidence/timestamp.vue submitApply`
**覆盖缺陷**：用户新增无响应、项目创建无提示、接口新增无响应、时间戳无响应
**置信度**：高｜**工作量**：小

### 任务 T8 — 【R8】修正租户隔离前后端路径不一致（404）
**根因**：前端 `views/tenant/isolationConfig.vue` 请求 `/tenant/isolation/config·status/list·test`、`dataIsolation.vue` 请求 `/tenant/dataIsolation/config`，后端 `TenantController` 只实现 `/tenant/getTenantIsolationConfig`、`/tenant/saveTenantIsolationConfig` → 404（顶部"请求出错(404)"）。
**改动**（二选一）：① 前端改指向已存在接口（推荐，页面已有 mock 兜底）；② 后端在 `TenantController` 补 `isolation/config`(GET/POST)、`isolation/status/list`、`isolation/test`、`dataIsolation/config`。
**覆盖缺陷**：租户间计算流程隔离 404、租户流程隔离 404
**置信度**：高｜**工作量**：小

### 任务 T9 — 【R9】修角色创建时间时区
**根因**：`SysRoleListVO.java:14` `@JsonFormat` 缺 `timezone`，容器 UTC 下回读差 8 小时；insert 依赖 DB 默认时间戳。
**改动**：`@JsonFormat(pattern="yyyy-MM-dd HH:mm:ss", timezone="GMT+8")`；根治可在 insert 显式写 `c_time=NOW()` 并统一 JDBC `serverTimezone`/`-Duser.timezone`。
**覆盖缺陷**：角色新增后创建时间与实时时间不一致
**置信度**：高｜**工作量**：极小

### 任务 T10 — 【R10】补齐后端占位/桩实现
**根因**：后端直接 `return success` 或写占位文本，不查库/不产文件。
**改动**：
- `ApiManageService.exportApiLog`（接口日志导出）：先查 count，无数据返业务码/`msg="无数据可导出"`，有数据写真实 CSV/Excel 流；前端 `views/api/log.vue exportLog` 按 blob/code 判定（当前仅 `if(res)` 恒真+`// TODO 下载`）。
- `DifferenceController.exportDifferenceLog`（求差日志导出）：当前仅写一行占位文本，改为查 ComputeLog 真数据导出（或复用 `/log/exportComputeLog`）。
- 计算日志记录导出乱码/空数据：确认部署包含提交 `c0b810b2`+`0c6dabca`（blob 识别 JSON 错误体 + 空数据提示）；如现网仍复现即为构建版本过旧。
**覆盖缺陷**：接口日志无数据却提示导出成功、计算日志记录空数据可导出且乱码、联邦求差日志导出
**置信度**：高/中｜**工作量**：中

### 任务 T11 — 【单独】节点管理创建机构失败
**根因**：`SysOrganService.java:155` 首次创建机构时**同步 HTTP 回调填入的网关** `/share/shareData/verifyGateway` 校验，新节点网关未就绪 → 失败/异常 → 前端"请求异常"。另 `:132-133` 存在对象混用隐患。
**改动**：部署时保证网关地址可达；或改为"先登记本地节点、延后校验网关"；修 `:132-133` 逻辑。
**覆盖缺陷**：节点管理创建机构请求异常
**置信度**：中高｜**工作量**：中

## 三、建议执行顺序（先低成本高覆盖）

1. **T4 + T5**（SQL：权限种子 + 建表）→ 一次数据库整改解锁大量"404/请求异常"，无需改代码重新构建
2. **T3**（网关路由）→ 解锁联邦学习整模块
3. **T1**（前端 returnCode→code 批量改）→ 一次前端小改修掉 7 条"成功却报错/查询失效"
4. **T7 + T9 + T8**（前端补分支 + 时区 + 路径）→ 低成本收尾
5. **T6**（字段契约）→ 修监控 6 条 + 时间戳 + 接口
6. **T2 + T10**（mock 接线 + 后端补实现）→ 工作量最大，后端已就绪，主要是前端接线
7. **T11**（节点/网关校验）→ 依赖部署环境

> 快速估算：T1/T3/T4/T5/T7/T8/T9 均为"小/极小"改动，合计可关闭约 **25/44** 缺陷；剩余主要落在 T2（前端接线约 20 页）与 T6/T10（契约+补实现）。
