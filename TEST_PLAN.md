## 一、测试架构

```
┌──────────────────────────────────────────────────────┐
│                  自动化测试框架                          │
├─────────────┬───────────────┬────────────────────────┤
│ 单元测试     │   集成测试     │    端到端测试(E2E)       │
│ (Jest)       │  (Python)     │  (Playwright)           │
│ Vue 组件     │  API 功能     │  浏览器模拟用户操作      │

```

## 二、测试工具

| 工具 | 用途 | 执行方式 |
|------|------|---------|
| Jest | Vue 组件单元测试 | `cd webconsole && npm run test:unit` |
| Python + Playwright | 浏览器 E2E 测试 | `python3 test_comprehensive.py` |
| Python + requests | API 接口测试 | `cd test-framework && pytest` |
| HTML 报告 | 测试结果可视化 | 自动生成到 `reports/` 目录 |

## 五、运行前置条件

### 5.1 前端静态测试 (无需后端)
```bash
# 1. 构建前端
cd primihub-webconsole && npm run build:prod

# 2. 启动HTTP服务
cd dist && python3 -m http.server 8080

# 3. 运行测试
python3 /tmp/test_comprehensive.py
```

### 5.2 全链路测试 (需要后端)
```bash
# 1. 启动后端服务
cd primihub-service && mvn spring-boot:run

# 2. 启动前端开发服务器
cd primihub-webconsole && npm run dev

# 3. 运行测试框架
cd primihub-test-framework && python3 run_tests.py
```

## 六、通过标准

所有测试项满足以下标准视为通过：
- **P0 (关键)**: 页面可正常加载，无白屏/崩溃 — **100% 必须通过**
- **P1 (重要)**: 核心交互（表单/按钮/导航）工作正常 — **≥90% 通过**
- **P2 (一般)**: 边缘交互、异常处理 — **≥80% 通过**
