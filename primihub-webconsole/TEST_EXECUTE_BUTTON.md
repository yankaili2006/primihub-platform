## 快速测试步骤

### 1. 启动开发环境

```bash
cd /mnt/data1/github/primihub-platform/primihub-webconsole
npm run dev
```

### 2. 打开浏览器测试

1. 打开浏览器并访问应用
2. 按 F12 打开开发者工具
3. 切换到 Console 标签

### 3. 测试执行按钮

#### 测试场景1：正常执行流程

**步骤**：
1. 导航到：项目管理 > 选择项目 > 项目详情
2. 点击"新建任务"或"创建任务"
3. 配置任务：
   - 选择模型类型
   - 添加数据集组件
   - 选择发起方和协作方数据
   - 配置模型参数
4. 点击顶部工具栏的"执行"按钮（播放图标）

**预期结果**：
- ✅ 控制台输出：
  ```
  ToolBar: runFn clicked, emitting run event
  TaskCanvas: run method called
  TaskCanvas: calling saveFn before run
  saveParams [...]
  TaskCanvas: saveFn completed, modelId: xxx
  TaskCanvas: validation passed, calling runTaskModel API
  TaskCanvas: runTaskModel response: {code: 0, result: {...}}
  ```
- ✅ 页面显示通知："开始运行"
- ✅ 1秒后自动跳转到任务详情页

#### 测试场景2：验证失败

**步骤**：
1. 创建任务但不选择数据集
2. 点击执行按钮

**预期结果**：
- ✅ 控制台输出：
  ```
  TaskCanvas: validation failed, aborting run
  ```
- ✅ 显示错误提示（例如："请选择发起方数据集"）

#### 测试场景3：网络错误模拟

**步骤**：
1. 打开开发者工具 > Network 标签
2. 勾选"Offline"模拟断网
3. 配置好任务后点击执行按钮

**预期结果**：
- ✅ 控制台输出错误：
  ```
  运行任务失败: [错误信息]
  ```
- ✅ 显示错误提示："运行任务失败，请稍后重试"

#### 测试场景4：保存失败

**步骤**：
1. 在 Network 标签中，找到 `saveModelAndComponent` 请求
2. 右键 > Block request URL
3. 配置任务后点击执行按钮

**预期结果**：
- ✅ 控制台输出：
  ```
  保存模型失败: [错误信息]
  运行任务异常: [错误信息]
  ```
- ✅ 显示错误提示："保存模型失败，请检查网络连接或稍后重试"

## 常见问题排查

### 问题1：点击按钮仍然无响应

**检查**：
1. 打开控制台，看是否有 `ToolBar: runFn clicked` 日志
   - 如果没有：检查按钮是否被禁用或被其他元素遮挡
   - 如果有：继续查看后续日志

2. 检查是否有 `TaskCanvas: run method called` 日志
   - 如果没有：检查事件绑定 `@run="run"`

3. 查看是否有错误日志
   - 如果有：根据错误信息定位问题

### 问题2：显示"模型运行中"

**原因**：`modelStartRun` 状态未正确重置

**解决**：
1. 刷新页面重试
2. 检查之前的任务是否还在运行

### 问题3：验证失败但没有提示

**检查**：
1. 查看控制台是否有 `validation failed` 日志
2. 检查 `checkRunValidated()` 方法的逻辑
3. 确认必填字段是否都已填写
